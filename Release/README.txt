================================================================================
PDF Font Changer - クロスプラットフォームビルドガイド
================================================================================

このフォルダには、Windows、macOS、Linux 向けのスタンドアローン実行ファイルを
ビルドするための設定ファイルと手順が含まれています。

================================================================================
【フォルダ構成】
================================================================================

Release/
├── README.txt              # このファイル
├── win_build.spec          # Windows版 PyInstaller設定ファイル
├── win_Readme.txt          # Windows版 詳細ビルド手順
├── mac_build.spec          # macOS版 PyInstaller設定ファイル
├── mac_Readme.txt          # macOS版 詳細ビルド手順
├── linux_build.spec        # Linux版 PyInstaller設定ファイル
└── linux_Readme.txt        # Linux版 詳細ビルド手順

================================================================================
【クイックスタート】
================================================================================

◆ Windows でビルドする場合：

   1. Release/win_Readme.txt を開く
   2. 手順に従ってビルド実行
   3. Release/dist/PDF Font Changer.exe が生成される

◆ macOS でビルドする場合：

   1. Release/mac_Readme.txt を開く
   2. 手順に従ってビルド実行
   3. Release/dist/PDF Font Changer.app が生成される

◆ Linux でビルドする場合：

   1. Release/linux_Readme.txt を開く
   2. 手順に従ってビルド実行
   3. Release/dist/pdf-font-changer が生成される

================================================================================
【共通の前提条件】
================================================================================

すべてのプラットフォームで以下が必要です：

1. Python 3.9 以上（3.12 推奨）
2. プロジェクトルートに requirements.txt の依存パッケージがインストール済み
3. PyInstaller がインストール済み（pip install pyinstaller）
4. assets/ フォルダにアイコンファイルが存在すること

================================================================================
【ビルドの仕組み】
================================================================================

このプロジェクトでは PyInstaller を使用しています。

PyInstaller は Python スクリプトを単一の実行ファイル（またはアプリバンドル）
に変換するツールで、以下の特徴があります：

✓ Python インタープリタを実行ファイルに埋め込み
✓ すべての依存ライブラリを同梱
✓ エンドユーザーは Python をインストール不要
✓ クロスプラットフォーム対応

【重要】
各プラットフォーム向けのビルドは、そのプラットフォーム上で実行する必要が
あります。つまり：

- Windows版は Windows でビルド
- macOS版は macOS でビルド
- Linux版は Linux でビルド

クロスコンパイルは基本的にサポートされていません。

================================================================================
【.spec ファイルについて】
================================================================================

各フォルダの .spec ファイルは PyInstaller の設定ファイルです。
以下の内容が定義されています：

- ソースファイルの場所（main.py, pdf_processor.py, font_scanner.py）
- 同梱するアセット（assets/ フォルダ）
- hiddenimports（自動検出されない依存モジュール）
- アイコンファイルの指定
- 実行ファイル名
- コンソール表示の有無
- UPX圧縮の有効/無効

.spec ファイルを編集することで、ビルド設定をカスタマイズできます。

================================================================================
【ビルド後の成果物】
================================================================================

◆ Windows版：
   → PDF Font Changer.exe（単一の実行ファイル）
   → ダブルクリックで起動可能
   → 50-80MB 程度

◆ macOS版：
   → PDF Font Changer.app（アプリケーションバンドル）
   → ダブルクリックまたは Applications フォルダにコピーして使用
   → 60-90MB 程度

◆ Linux版：
   → PDF Font Changer（単一の実行ファイル）
   → ターミナルから ./pdf-font-changer で起動
   → 60-90MB 程度

※ サイズは含まれるライブラリや圧縮設定により変動します

================================================================================
【配布方法】
================================================================================

生成された実行ファイル/アプリは、そのままエンドユーザーに配布できます。

◆ Windows：
   - ZIP圧縮して配布
   - インストーラー（Inno Setup 等）に組み込むことも可能

◆ macOS：
   - DMGファイルに梱包して配布（推奨）
   - ZIP圧縮して配布も可
   - コード署名・公証を行うとよりスムーズ

◆ Linux：
   - tar.gz で圧縮して配布
   - AppImage 形式に変換（推奨）
   - Flatpak / Snap パッケージ化も可能

================================================================================
【トラブルシューティング】
================================================================================

◆ ビルドが失敗する場合：

   1. Python バージョンを確認（3.9以上、3.12推奨）
   2. 仮想環境を使用しているか確認
   3. requirements.txt のパッケージがすべてインストール済みか確認
   4. キャッシュをクリア：
      # Windows:
      rmdir /s /q build dist __pycache__
      # macOS/Linux:
      rm -rf build dist __pycache__

◆ 実行ファイルが起動しない場合：

   1. console=False を console=True に変更して再ビルド
   2. エラーメッセージを確認してデバッグ
   3. 各プラットフォームの Readme（win_Readme.txt / mac_Readme.txt / linux_Readme.txt）のトラブルシューティングを参照

◆ ファイルサイズが大きすぎる場合：

   1. UPX圧縮を有効化（upx=True）
   2. 不要な hiddenimports を削除
   3. excludes リストに不要なモジュールを追加

================================================================================
【カスタマイズ】
================================================================================

各 .spec ファイルを編集することで、以下をカスタマイズできます：

1. アプリケーション名の変更
2. アイコンの変更
3. バージョン情報の変更
4. 同梱するファイルの追加/削除
5. コンパイルオプションの調整

編集後は pyinstaller build_xxx.spec を再実行してください。

================================================================================
【継続的インテグレーション（CI）】
================================================================================

GitHub Actions 等の CI サービスを使用すると、コミット時に自動ビルドが可能です。

参考例：

- Windows: windows-latest ランナーを使用
- macOS: macos-latest ランナーを使用
- Linux: ubuntu-latest ランナーを使用

各ランナーで該当する .spec ファイルを使ってビルドし、成果物をアーティファクト
として保存できます。

詳細は GitHub Actions のドキュメントを参照してください。
